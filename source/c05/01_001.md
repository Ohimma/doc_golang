## 1.001 算法: 基础知识

#### 1. 时空复杂度

```
O后面的函数，指明某个算法的耗时/耗空间与数据增长量之间的关系。其中的n代表输入数据的量

1. O(1)
最低的时空复杂度，即耗时与数据多少大小无关，哈希算法就是典型的O(1)代表。
无论数据规模多大，都可以在一次计算后找到目标。冲突的话就很麻烦，指向的value会做二次hash到另一块存储区域。

举例：我是一个酒店管理员，管理一百个房间钥匙，我把每个钥匙放到抽屉里，并标上编号，这样一次就能找到任意钥匙了

2. O(n)
数据量增大几倍，耗时也就增大几倍，线性。比如常见的遍历算法
要找到一个数组里面最大的一个数，你要把n个变量都扫描一遍，操作次数为n，那么算法复杂度是O(n).

3. O(n^2) 
代表数据量增大n倍时，耗时增大n的平方倍，复杂度高于线性 平方。比如冒泡排序。 
用冒泡排序排一个数组，对于n个变量的数组，需要交换变量位置次，那么算法复杂度就是O().

4. O(log n)
代表数据增大n倍时，耗时增大log n倍，比如当数据增大256倍时，耗时增大8倍。常见的二分查找

解释：log是对数函数，即10的2次方等于100，则10就是以2为底的100的对数

5. O(n log n)
就是n乘以log n，当数据增大256倍时，耗时增大256*8倍。复杂度高于线性低于平方。常见的归并排序
```

#### 2. 数据结构

```
线性表结构：数据排成像一条线一样的结构。数组、链表、队列、栈
1. 数组
特点：一组连续的内存空间、存储相同数据类型的数据
优点：查询快、遍历方便；
缺点：大小固定无法扩容、只能存储一种类型、添加删除比较慢(因为要移动)
场景：适用查询频繁，对存储要求较高，很少增加删除操作的情况

2. 栈
特点：特殊线性表，先入后出
场景：适用于递归场景，比如斐波那契数列

3. 队列
特点：特殊线性表，先入先出
场景：多线程阻塞队列管理中很适用

4. 链表
特点：存储非连续非顺序，元素包含数据域和指针域，数据通过指针找到下一元素
优点：不需要初始化，可任意加减元素，添加删除只需要改变前后两个元素的指针即可
缺点：有大量指针域，占用空间大、查找元素需要遍历，较慢
场景：适用数据量较小，需频繁增加和删除的场景



非线性表结构：数据并不是简单的前后关系。二叉树、堆、图
1. 树
特点：由n(n>=1)个有限节点组成的具有层次关系的集合，mysql索引用了B+树，哈希用了红黑树
优势：既有链表好处，也有数组的好处，是两者的结合
场景：处理大批量的动态数据方面非常有用

2. 哈希(散列表)
特点：根据key和value直接进行访问的数据结构。


3. 堆
特点：一棵树的数组对象，对中某个节点的值总是不大于或不小于其父节点的值，堆总是一颗完全二叉树。
场景：因为堆有序的特点，总用来做数组中的排序

4. 图
特点：由结点的有穷集合v和边的集合e组成。为了与树加以区别，常常将结点成为顶点，边是顶点的有序偶对。


```
